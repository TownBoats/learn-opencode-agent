# 中级篇导读

## 适合谁读

- 已经阅读完理论篇与实践篇基础章节，理解 Agent 的核心组件、Prompt 域与简单工具链，并期待构建更健壮的工程解决方案；
- 正在经历 RAG 召回不准、生成结果不稳定、多 Agent 协作或 Planning 设计反复改版的真实项目，需要与团队共享标准流程与判断；
- 希望在生产、成本、安全等长期工程维度中梳理引用路径与分工边界，不愿意仅停留在单次 Prompt 实验。

## 首批推荐阅读顺序

1. RAG 五大翻车场景（25-RAG）
2. 多智能体协作（26-Multi-Agent）
3. Planning 机制（27-Planning）
4. 上下文工程实战（28-Context Engineering）
5. System Prompt 设计指南（29-System Prompt）
6. 生产架构（30-Production Architecture）
7. 安全与边界（31-Safety Boundaries）
8. 性能与成本优化（32-Performance & Cost）

这些章节由浅入深，由问题驱动，先抓取核心困境，再逐步过渡到生产化考量。

## 阅读建议

- **从 RAG / 多智能体 / Planning 开始**：先从经典的稳定性、协作、计划三角展开，借助已有章节 P7-P11 的理论基础，快速锁定最常见的工程瓶颈再逐项打磨。 |
- **从生产 / 安全 / 成本开始**：如果当前挑战聚焦落地质量，从生产架构与安全边界出发，回链到部署、工具与成本管理章节，将中级篇作为架构决策的检查表。 |

- 推荐结合上下文工程与 System Prompt 两章的内容，补齐“上下文 + 指令”这两条线。 |
- 每篇内容都附带源码映射和教学示例，阅读时可根据自己的任务背景选择深入源码还是运行示例。
