# 首页优化设计文档

**日期**: 2026-03-21
**范围**: `docs/book/docs/index.md`
**不涉及**: 任何 Vue 组件文件

---

## 背景与问题

当前首页存在以下问题：

1. **Hero 按钮过多**：6 个按钮（brand×2 + alt×4），用户注意力分散，主 CTA 不突出
2. **内容冗余**："这三条线分别回答什么"与"学习方式"表达几乎相同的信息；"这本书怎么读"与 `<LearningPath>` 组件高度重叠
3. **章节顺序失衡**："你会得到什么"和"适合人群"是帮助新访客判断是否值得继续阅读的关键信息，却排在最底部
4. **阅读边界 blockquote** 插在正文中间打断阅读节奏

---

## 方案选择

采用**方案 B（逻辑重排）**：不修改任何 Vue 组件，只调整 `index.md` 的内容结构。

---

## 改动详情

### 改动 1：Hero 按钮从 6 个精简为 3 个

**保留**（主 CTA）：
- `开始阅读理论篇`（brand）
- `动手实践 23 个项目`（brand）
- `阅读地图`（alt）

**移除**（移到底部辅助入口区）：
- `阅读中级篇`
- `Star 支持本书`
- `查看源码基线`

### 改动 2：合并冗余段落

将"这三条线分别回答什么"和"学习方式"两个 H2 段落合并为一个精简段落，标题改为"这本书解决什么问题"。

**保留**：
- "这三条线"中的三条核心定位（理论篇/实践篇/中级篇各一句）
- "学习方式"中关于"理论与实践穿插学习"的建议一句（例：建议理论与实践穿插学习，看完 1-4 章后先跑通 P1-P4）

**删除**：
- "学习方式"中与三条线重叠的 bullet list（三篇章节列举）
- "学习方式"中关于中级篇的跳转提示（中级篇入口移至底部）

### 改动 3：上移价值主张

将"你会得到什么"和"适合人群"两节从页面底部移动到合并后的定位说明之后、`<PracticePreview>` 之前。

### 改动 4：删除"这本书怎么读"

该节的 4 条阅读路径建议与 `<LearningPath>` 组件内容重叠，删除后由组件承载，不丢失信息。

### 改动 5：底部辅助入口扩充

在现有"辅助阅读入口"节（阅读地图/版本说明/术语表/封版清单）下方追加，顺序如下：

1. `阅读中级篇` 链接 → 沿用 Hero 中的 href：`/intermediate/`
2. `Star 支持本书` 链接 → 沿用 Hero 中的 href：`https://github.com/qqzhangyanhua/learn-opencode-agent`
3. `查看源码基线` 链接 → 沿用 Hero 中的 href：`https://github.com/anomalyco/opencode/tree/f8475649da1cd7a6d49f8f30ee2fad374c2f4fcc`
4. "阅读边界" blockquote（最后，置于所有链接之后）

## 新首页结构

```
Hero
  └─ 书名 / tagline / 3 个按钮（理论 + 实践 + 阅读地图）

① 这本书解决什么问题（精简合并段）
② 你会得到什么
③ 适合人群
④ 双轨学习体系（<PracticePreview>）
⑤ 核心学习路径（<LearningPath>）
⑥ 先记住这条主链路（<RuntimeLifecycleDiagram>）
⑦ 技术栈（<TechStackGrid>）
⑧ 辅助阅读入口（底部，包含中级篇 + Star + 源码基线 + 阅读边界）
```

---

## 约束

- 只修改 `docs/index.md`，不改任何 `.vue` 文件或 `config.mts`
- 保留所有现有 Vue 组件引用（import + 使用）不变
- 保留 frontmatter 不变（`layout: home` 等）
- 所有移除的链接在底部辅助入口区保留入口，不丢失信息
